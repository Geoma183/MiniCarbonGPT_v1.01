# Upload Checklist

Use this checklist before updating the public GitHub repository.

- [ ] Confirm no API keys or private credentials are present.
- [ ] Confirm local paths are only in `configs/local_config.yaml`, which is ignored by Git.
- [ ] Confirm restricted full-text papers are not committed.
- [ ] Confirm benchmark files are redistributable, or replace them with samples and metadata.
- [ ] Confirm `results/` contains aggregate evidence tables rather than private raw logs.
- [ ] Confirm README points to the correct manuscript and revision status.
- [ ] Create a GitHub release or tag such as `v1.01-envsoft-revision`.
- [ ] Cite the exact repository release/tag in the revised manuscript and response letter.
